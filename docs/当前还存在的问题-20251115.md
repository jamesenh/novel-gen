# 当前工具还需要优化的(根据生成的小说结果评估出来的)

### 1. 语言 & 视角没写死

**问题：**

* 没明确要求用中文
* 没明确人称/视角（容易乱切心声）

**解决方向：**

* 在 system 里明确：使用「简体中文」
* 指定叙述视角（比如「第三人称有限视角，以本场景核心角色为视角」）

---

### 2. 字数要求不够具体

**问题：**

* 只说“误差不超过20%”，但没说明字数来源、单位
* `estimated_words` 没在提示里明确重申

**解决方向：**

* 在 system 文案里写清：字数以 `scene_plan.estimated_words` 为基准，单位是中文字符数，允许±20%
* 在 user 段再明确写一遍目标字数和可接受范围

---

### 3. 节奏要求太抽象

**问题：**

* “节奏要把握好，不拖沓不急躁” 属于空话
* 模型不知道当前场景是高潮/缓冲/日常

**解决方向：**

* 在场景计划中增加类型字段（例如 scene_type/intensity）
* 在 system 中根据这个字段给出具体节奏指令（例如：战斗场景少环境多动作，对话场景多心理描写等）

---

### 4. 前文概要的作用不清晰

**问题：**

* 没说是仅供参考，还是需要复述
* 容易出现重复讲前文，或干脆无视前文

**解决方向：**

* 明确说明：前文概要用于保持连贯性和设定一致，**不要在正文中机械复述**
* 要求模型在角色关系、境界、事件承接上参考前文概要，避免矛盾

---

### 5. 容易产生套路化 & 描述重复

**问题：**

* 没有任何提示让模型避免反复使用相同的比喻/外貌描写
* 人物出场时容易复制粘贴同一段描述

**解决方向：**

* 在 system 中增加要求：避免陈词滥调和重复描写，同一人物反复出场时应变换描写角度
* 后续可以考虑将高频禁用描述传入 prompt（这一条留个口子）

---

### 6. 格式说明有点重复/模糊

**问题：**

* system 文案里有自己的输出说明，同时又插入 `{format_instructions}`
* 容易让模型混淆以谁为准

**解决方向：**

* 把 JSON 结构和格式要求统一交给 `{format_instructions}`
* system 只强调：必须严格按其输出，不要 Markdown，不要额外文字

---

### 7. “符合世界观设定和角色性格” 没有硬约束

**问题：**

* 只是口头说要符合设定，没有说“不得修改设定 / 不得改变既有关系”
* 容易在长篇里悄悄改设定

**解决方向：**

* 在 system 里强调：不得更改 world/characters 中已有信息，只能在其基础上展开
* 遇到不确定的地方倾向于模糊处理，而不是自己编新设定
