## Why

当前 `ng chat` 的自然语言意图识别为关键词版本：只要输入包含“生成”等动词，就会被归类为“生成/继续生成”，并直接触发 `workflow.run`/`workflow.resume` 执行完整工作流。这会导致用户仅想“生成某个阶段产物”（如“生成人物角色”）时，系统仍然继续执行后续节点（大纲、章节计划、正文等），产生不符合预期的行为与额外成本。

本提案旨在把对话式 Agent 的“生成”交互从“动词触发全流程”改为“目标驱动的阶段执行”：用户说“生成 X”，系统只生成到对应节点为止；若缺少前置依赖，先询问是否允许补齐前置，再执行并在目标节点停住。

## What Changes

- **目标型生成路由**：在自然语言生成意图中识别“生成目标/产物”（世界观/主题冲突/人物角色/大纲等），映射到工作流节点名，并用 `stop_at=<node>` 定向执行到目标节点。
- **前置补齐需显式同意**：当目标节点所需前置（如人物角色依赖世界观、主题冲突）缺失时，系统先展示“将补齐哪些前置 + 将停在哪个节点”的计划并请求用户确认；用户拒绝则不执行，并给出替代建议。
- **避免误触发全流程**：仅当用户表达“一键生成/开始生成/继续生成/跑完整流程”等全流程意图（或使用 `/run`、`/resume`）时才触发完整工作流；“生成 X”默认视为目标型生成。
- **测试与文档同步**：补充 `ng chat` 自然语言示例与回归用例，覆盖“生成人物角色”在不同项目状态下的行为。

## Impact

- **Dependencies**
  - 本变更建立在 `agent-chat` 能力之上（如尚未落入主 specs，可与 `add-chat-agent-kuzu` 一并评审/落盘）。

- **Affected specs（通过 spec deltas 更新）**
  - `agent-chat`：补充“目标型生成 + 前置补齐确认 + stop_at 执行”的要求与场景
  - `orchestration`：明确 `run_workflow(stop_at=...)` 的暂停语义与约束（为对话式定向执行提供编排层契约）

- **Affected code（实现阶段将改动/新增）**
  - `novelgen/agent/chat.py`：目标抽取、前置依赖检查、确认与计划执行
  - `novelgen/tools/workflow_tools.py`：在 ToolPlan 中透传 `stop_at`（如需补齐）及相关输出信息格式化
  - `docs/对话式Agent使用指南.md`：补充目标型生成的行为说明与示例

## Non-Goals（本提案不做）

- 不引入 LLM 分类器（仍保持规则/关键词为主的路由策略）
- 不重构 LangGraph 工作流拓扑或节点依赖关系（仅调整对话层的执行策略与停点）
- 不实现“强制重算/按阶段 force 覆盖”的完整交互（可作为后续提案扩展）
